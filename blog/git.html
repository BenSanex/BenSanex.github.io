First Post!

Introducing Command Line and Git

2/12/16

Hello all! I've never written any sort of blog before, so this is the very first blog post I've ever done. This past week I've learned a lot of concepts that are brand new to me even though I am not brand new to programming. That said, using the command line and the idea of version control are clearly very important tools that developers use when programming.

Though I've used linux on multiple different computers in the past, I have little experience actually using and understanding the command line interface. In addition, I wasn't sure how it was more beneficial to developers than just using a GUI based file system. As I am beginning to learn, the command line is a much cleaner and faster way to manipulate and work with the files and commands that though possible, are slower using a GUI. Using hotkeys and typing commands seems to always be faster than reaching for a mouse. Even in every day use, commands like ALT+Tab or CTRL+T to navigate your user interface can save a lot of time, and this is extended when you are able to maniuplate your filesystem without using a mouse.

The command line opens up something very important to developers, which is version control. In the past I have tried to make a website and seen that trying to keep track of the changes I make while testing them even on a small scale quickly gets out of hand. I found myself duplicating files over and over, and trying to use CTRL+Z repeatedly to try to undo changes that would adversly affect my whole site. With Git, all of your changes are tracked. You "commit" checkpoints which you can easily roll back to if you make a mistake. Any time you make a significant change or set of changes and are ready to test, you commit the changes and then have the ability to go to the live environment and test them. If they are changes you like, you then have the ability to merge them with the known working version of your code and update that to be your latest working version.

GitHub is a cloud service that interfaces with Git on your local computer to store code. It allows you to keep a copy in the cloud just in case your computer gets tossed out a window or is set on fire. This also allows for easier collaboration. By default all of your code is visible to the public. I won't go too much into the benefits of collaboration, but GitHub makes it very practical.

Although I haven't been learning programming languages, the knowledge I gained over the past week was eye opening. Before starting at DBC, I always said I didn't know what I didn't know, but now I do. There are so many tools and best practices developers use, and version control is perhaps the most important thing I was never aware of before.
